# Review Jira Backlog Items

Review the user's backlog items to assess whether each should remain in backlog or move to a different status.

## Instructions

1. Query Jira for the user's backlog items
2. For each item: read the description, recent comments, and linked items
3. Research whether the item should stay in backlog or move (e.g., ready for work, blocked, stale, closeable)
4. Produce a summary report with recommendations per item

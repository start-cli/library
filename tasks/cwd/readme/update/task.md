# Update README.md

Update an existing README.md for the current working directory. A human-focused document that helps people understand, install, and use the project. The directory may be a repository, a package subdirectory, a standalone tool, or any other project structure.

## Instructions

1. Confirm a README.md file exists in the current working directory
2. Follow the process below to update it

## Process

Update an existing README.md by:

1. Read the current README.md to understand its structure and content
2. Explore the directory for changes that may affect the documentation
3. Identify outdated, missing, or incorrect information
4. Update sections based on the user's request or discovered changes

Preserve any project-specific customisations and tone.

Modes:

- Targeted: focus on the section or topic requested by the user
- Refresh: review the entire document against the current state of the directory

## Style Guidelines

README.md is a human-facing document. Use full CommonMark as appropriate:

- Use a descriptive project title as the main heading, not "README" or "README.md"
- Keep the description concise - one to three sentences that explain what the project does and who it is for
- Use badges sparingly and only where they add value (build status, version, license)
- Prefer real command examples over abstract descriptions
- Use code blocks with language identifiers for all code and commands
- Include a license section that names the licence and links to the LICENSE file
- Avoid filler sections with placeholder text - only include a section if you have content for it
- Keep contributing and license sections brief; link to separate files where they exist
- Do not remove sections without confirming with the user

# Create AGENTS.md

Create an AGENTS.md document for a repository. A README for AI agents with context and instructions for repository work.

## Instructions

1. Fetch the AGENTS.md specification from https://agents.md/ for context and format
2. Confirm no AGENTS.md file exists in the repository root
3. Follow the process below to create one

## Process

Create a new AGENTS.md by:

1. Discover and read existing documentation
2. Explore the repository to understand the technology stack
3. Identify build systems, test frameworks, and development workflows
4. Find code style configurations (linters, formatters)
5. Create a comprehensive AGENTS.md covering sections relevant to the repository

Focus on information agents need that may not be obvious from the codebase:

- Exact commands to build, test, and validate
- Project-specific conventions and patterns
- Common pitfalls and their solutions
- Environment setup requirements

## Restrictions

When writing the AGENTS.md document, do not use:

- Bold or italic formatting
- Horizontal rules
- Emojis
- HTML comments
- Image embeds (use regular links)
- Multiple consecutive blank lines
- Nested lists beyond 3 levels
- Task lists
- Heading depth beyond `###`
- Directory structures beyond depth 3

Keep:

- Headings for structure
- Single blank lines between sections
- Inline code and code blocks
- Tables for structured data
- Lists (ordered and unordered)
- Callout prefixes without bold

## Style Guidelines

- Use a descriptive project title as the main heading, not "AGENTS.md"
- Avoid repeating information across sections; consolidate or cross-reference
- Keep table columns consistent in purpose
